WEATHER APP

Weather app est une mini application qui m'a servi de test d'aptitude. Elle devait remplir les conditions suivantes :

  • une jauge doit se remplir à 100% en 60sec.
  • toute les dix secondes l'application doit faire un appel météo pour cinq villes:
    1. 0 sec: Rennes
    2. 10 sec: Paris
    3. 20 sec: Nantes
    4. 30 sec: Bordeaux
    5. 40 sec: Lyon

  • un message d'attente est affiché pour faire patienter l'utilisateur toutes les six secondes:
    1. Nous téléchargeons les données...
    2. C'est presque fini...
    3. Plus que quelques secondes avant d'avoir le résultat...

  • Lorsque la jauge est remplie on affiche un tableau de cinq lignes qui donnera le nom de chaques villes ainsi que leurs température et le temps sous forme d'image.
  • La jauge se transforme en bouton "recommencer".
  • il est possible de faire un back pour revenir à l'écran d'accueil.

Caractéristiques:

  • Langage: Kotlin
  • Architecture: MVVM
  • API/Librairies:
    • Dagger-Hilt
    • Retrofit
    • Open Weather Map
    • Glide

Le repository est disponible sur GitHub ici

DÉBUT DU TRAVAIL

Une fois que l'utilisateur appuie sur le bouton "VOIR LA MÉTÉO", il arrive sur un nouvel écran où une jauge qui commence à se remplir en affichant divers messages. Pendant ce temps l'application télécharge la météo pour les différentes villes toutes les dix secondes.

FIN DU TRAVAIL

Une fois que la jauge atteint les cents pourcents, un tableau est affiché et montre le nom de chaques villes avec leur température et le temps actuel.
La jauge est remplacée par un bouton "RECOMMENCER"
